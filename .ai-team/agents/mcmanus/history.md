- **SDK Replatforming: User Impact Analysis (2026-02-21)** â€” Brady asked: "Once we change, will users lose anything?" Analyzed user-facing implications of migrating from prompt-only coordinator to SDK-backed hybrid (v0.6.0) and full Node.js process (v0.7.0). **Day-One Changes: MINIMAL AND BACKWARD-COMPATIBLE.** Users keeping `.squad/` (or `.ai-team/`) get all existing features unchanged. New `squad orchestrate` subcommand is opt-in. Install via `npx github:bradygaster/squad` stays identical. Deprecation warnings appear only if using legacy `.ai-team/` (announced in v0.5.0). **What Users KEEP:** (1) One-command install (zero new dependencies until v0.7.0, v0.6.0 adds optional SDK for new `orchestrate` command only). (2) Existing `.squad/` folder structure, decisions.md, agent history, casting universe â€” all untouched. (3) Current `npx github:bradygaster/squad upgrade` workflow. (4) All documented workflows (GitHub Issues Mode, Ralph, Copilot coding agent, team growth). (5) Existing agents continue working unchanged via prompt-only coordinator. (6) No retraining â€” agents' history.md knowledge doesn't change format or location. **What Users GAIN:** (1) Silent success bug eliminated (v0.7.0 uses SDK's sendAndWait with timeout). (2) Crash recovery (resumeSession for incomplete work). (3) Multi-session management (future: checkpoint-driven replay). (4) Real-time agent status visibility. (5) Optional: per-agent MCP server isolation (advanced feature). **Migration Friction Points:** (1) Zero for v0.6.0 (hybrid, all existing code works). (2) Minimal for v0.7.0 (SDK dependencies added, but install already runs npm, just bigger node_modules). (3) Messaging burden: users need to understand "v0.6 is hybrid, v0.7 is full SDK" â€” requires ONE clear changelog entry and one blog post. (4) **The escape hatch:** If SDK breaks, users can pin v0.5.2 forever (teams are git-stored, fully portable, Squad is npm package). **Risk if SDK goes away:** Low. Worst case: fork SDK (MIT licensed), Squad code can fall back to prompt-only coordinator (v0.5.2 coordinator still exists). Users would never need to migrate away; they'd just pin to the last good version with their team intact. **Bottom line:** This is NOT the `.ai-team/` â†’ `.squad/` friction of v0.5.0 (which asked users to run migration). v0.6.0/v0.7.0 is opt-in infrastructure change, existing users see zero difference, new users benefit from better reliability and UX. Safe to ship. **Key learning:** SDK adoption is safe because Squad's core (agent files, decisions, casting, team state) is deliberately platform-agnostic. We can replatform the ORCHESTRATION layer without touching the MEMORY layer. That's the moat.
